MxFrameRateDevice


MxFrameRateDevice is used by Mx to hold a continuously differentiable time function.  


That function (or object) is here referred to as a frame rate device.


A frame rate device is a function/object that responds to .value(beat) by returning a value [usually a float]


Examples are SplineFr which evaluates a Spline along the time domain [x] and returns the y value for that point in time.  ie. splines for automation curves


Any continuously differentiable function can be a frame rate device.


A device is registered with the Mx using addFrameRateDevice


An MxUnit driver file can specify frame rate devices to be added by implementing a handler method:


frameRateDevices: {

[ funcOrObject ]

}


see drivers/SplineFr.scd


Mx calls tick(time) on all of its registered devices at whatever frame rate the Mx is set to.


the object/function then gets .changed(value) called on it 

[maybe this is an unclear design. the object didn't really change at all and it doesn't store its own value. maybe use notification system instead]


frame rate devices like SplineFr have an output type of:

MxSendsValueOnChanged


so attached inputs will respond to the changed/update notification and set value on the synth or do whatever it is that they want to do with the new value.
